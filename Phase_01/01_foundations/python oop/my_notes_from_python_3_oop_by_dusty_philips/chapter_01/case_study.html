Polymorphism in action
// Catalog শুধু LibraryItem জানে for (LibraryItem item : catalog.search("Humayun Ahmed")) { // Book হোক, DVD হোক — সবাই নিজের মতো locate() দেয় System.out.println(item.locate()); }
💡 Inheritance মানে — parent-এর সব কিছু পাওয়া। Polymorphism মানে — একই method call করলে, প্রতিটা child নিজের মতো করে respond করে।
// sequence diagram — search flow
1
User → Catalog search("Humayun Ahmed")
User search bar থেকে author নাম দিয়ে request পাঠায়
2
Catalog → LibraryItem[ ] filter by contributor
Catalog তার item list loop করে matching items খোঁজে
3
Catalog → each Item locate()
প্রতিটা matching item-কে locate() call করে — Book তার DDS দেয়, DVD তার Genre দেয়
4
Item[ ] → Catalog location: String
Items তাদের location return করে Catalog-এ
5
Catalog → User results[ ]
Catalog সব results একসাথে User-কে দেখায়
💡 Class Diagram বলে "কে কে আছে" — Sequence Diagram বলে "কে কার সাথে কীভাবে কথা বলে"। দুটো মিলে পুরো picture পাওয়া যায়।
❌ Bad — Inheritance

Spaghetti Design

// আলাদা subclass প্রতি role-এর জন্য class Author extends Contributor { } class Actor extends Contributor { } class Director extends Contributor { } class Artist extends Contributor { } // নতুন role? → নতুন class বানাও!
🍝 নতুন role আসলে নতুন class। Class বাড়তেই থাকে। Maintain করা nightmare।
✅ Good — Composition

Clean Design

// একটাই class, type দিয়ে role ঠিক হয় class ContributorWithType { Contributor person; String type; // "Author","Actor"... } // নতুন role? শুধু নতুন String!
✨ নতুন role = নতুন String value। Class structure change করতে হয় না।
// composition relationship
📚 Book
has many →
🔗 ContributorWithType
links to →
👤 Contributor
type = "Author" type = "Actor" type = "Director" type = "Artist"
💡 Refactoring মানে বাইরে থেকে কাজ same, কিন্তু ভেতরের design পরিষ্কার। Composition বলে — "object ধরো, extend করো না।" এটাই OOP-এর real mastery।
